#### 解决的BUG & 问题
+ url的bug，图片不展示（已经知道了，是http明文问题，需要设置权限，可以选择全局允许、也可以选择具体域名）

```xml
<!-- res/xml/network_security_config.xml -->
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <!-- 允许所有HTTP明文传输 -->
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
</network-security-config>


<!-- res/xml/network_security_config.xml -->
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <!-- 允许所有HTTP明文传输（仅调试用） -->
    <domain-config cleartextTrafficPermitted="true">
        <domain includeSubdomains="true">putv.qpic.cn</domain>
        <domain includeSubdomains="true">其他需要允许的域名</domain>
    </domain-config>
</network-security-config>
```

#### 代码规范(又去完整的阅读了一下代码规范，不只靠检查脚本，其实检查脚本我并没有用上)
记一下自己没意识到的：
+ 单个源文件最大长度2000行,自动生成的代码除外
+ 对从来没有用到的或者被注释的方法、变量、类、配置文件、动态配置属性等要坚决从系统中清理出去，避免造成过多垃圾及干扰性。
+ 对于受保护的文件建议在文件起始位置添加相应版权声明。
+ 所有的静态导入独立成组，非静态导入第三方的包，字典序。例如：com, junit, org, sun
+ long应使用大写字母L后缀，十六进制应使用大写字母A-F，所有其他数字前缀，中缀和后缀均应使用小写字母
+ 每次只声明一个变量,不要使用组合声明，比如int a, b;
+ 单个的注解可以和方法签名的第一行出现在同一行;
+ 【必须】lambda优于匿名类，对于函数式接口必须使用Lambda表达式（而不是匿名类）, 例如Runnable接口；（等等，很多条，可以现看）
+ 类和成员的修饰符如果存在，则按Java语言规范中推荐的顺序出现。（public protected private abstract static final transient volatile synchronized native strictfp）
+ 不要在finally块中使用return，finally块中的return返回后方法结束执行，不会再返回try块中的return语句的结果，即返回值被 finally 的返回值覆盖。
+ 编程实践 
    + 正则表达式需预编译
    + 需要 Map 的主键和取值时，应该迭代 entrySet()
    ```java
    Map<String, String> map = ...;
    for (Map.Entry<String, String> entry : map.entrySet()) {
        String key = entry.getKey();
        String value = entry.getValue();
        ...
    }
    ```
    + 不要使用魔法值
    + 建议使用 try-with-resources 语句
    ```java
    private void handle(String fileName) {
        try (BufferedReader reader = new BufferedReader(new FileReader(fileName))) {
            String line;
            while ((line = reader.readLine()) != null) {
                ...
            }
        } catch (Exception e) {
            ...
        }
    }
    ```
