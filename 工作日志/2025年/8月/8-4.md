### websocket和http

WebSocket是一种在单个TCP连接上进行全双工通信的协议，允许服务器和客户端之间实时交换数据。以下是WebSocket与HTTP的主要异同点：

异同点 
+ 连接方式：
    + HTTP：每次请求需要重新建立连接，是无状态的请求-响应模型。
    + WebSocket：建立一次连接后可以持久化，支持双向通信。
+ 通信模式：
    + HTTP：客户端发起请求，服务器响应，通信是单向的。
    + WebSocket：服务器和客户端可以随时主动发送数据，通信是双向的。
+ 性能：
    + HTTP：频繁的连接建立和关闭会增加开销。
    + WebSocket：减少了连接建立的开销，适合实时性要求高的场景。

OkHttp涉及WebSocket的原因
OkHttp是一个功能强大的HTTP客户端库，支持HTTP/1.1、HTTP/2和WebSocket协议。引入WebSocket支持的原因包括：
+ 扩展功能：支持实时通信场景（如聊天应用、实时数据推送等）。
+ 灵活性：通过forWebSocket参数区分普通HTTP请求和WebSocket请求，确保网络拦截器不会干扰WebSocket协议（如你提供的代码所示）。
+ 统一接口：开发者可以使用同一套API处理HTTP和WebSocket通信，简化开发流程。


### 一些概念
+ Socket（套接字）：
    + 网络通信的基本抽象，表示一个通信端点
    + 在Java中通过java.net.Socket类实现
    + 包含输入流(InputStream)和输出流(OutputStream)
    + OkHttp中的RealConnection类直接使用Socket进行TCP通信

+ SSL/TLS：
    + 安全套接字层/传输层安全协议
    + 用于加密网络通信，防止窃听和篡改
    + 在OkHttp中通过SSLSocketFactory创建安全套接字
    + 涉及证书验证、密钥交换等安全机制

+ Sink（输出流）：
    + OkIO库中的概念，对应Java的OutputStream
    + 表示数据输出的抽象
    + 在OkHttp中用于写入HTTP请求数据
    + 通过BufferedSink提供缓冲功能提高性能

+ Source（输入流）：
    + OkIO库中的概念，对应Java的InputStream
    + 表示数据输入的抽象
    + 在OkHttp中用于读取HTTP响应数据
    + 通过BufferedSource提供缓冲功能

+ HTTP Codec（编解码器）：
    + 负责HTTP协议数据的编码和解码
    + Http1Codec处理HTTP/1.1协议
    + Http2Codec处理HTTP/2协议
    + 建立在Socket和Sink/Source之上
