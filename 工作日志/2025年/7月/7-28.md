#### volley
Volley是Google为Android平台开发的一个轻量级HTTP库，主要用于简化网络请求的处理。它特别适合用于数据量较小但频繁的网络请求场景，例如加载图片或获取JSON数据。Volley的主要特点包括：
+ 自动调度请求：Volley会自动管理请求队列，优化网络请求的执行顺序。
+ 缓存支持：内置缓存机制，减少重复请求的开销。
+ 可扩展性：支持自定义请求类型和响应解析。
+ 主线程友好：默认情况下，响应回调会在主线程执行，方便更新UI。

#### okhttp or okhttp3的区别：

1. API 设计改进
    + 更简洁的API：okhttp3对API进行了重构，使其更加简洁和易用。例如，Request和Response类的设计更加模块化。
    + 链式调用：okhttp3引入了更多的链式调用（Builder模式），使得代码更易读和编写。
2. 性能优化
    + 连接池管理：okhttp3优化了连接池的管理，减少了连接建立的开销，提高了复用率。
    + HTTP/2支持：okhttp3原生支持HTTP/2协议，提供了多路复用和头部压缩等功能，显著提升了网络性能。
    + 更高效的缓存机制：okhttp3改进了缓存策略，支持更灵活的缓存控制。
3. 功能扩展
    + 拦截器（Interceptors）：okhttp3引入了拦截器机制，允许开发者在请求和响应的处理链中插入自定义逻辑（如日志记录、重试机制等）。
    + WebSocket支持：okhttp3增加了对WebSocket协议的支持，而okhttp没有这一功能。
    + 更强大的证书验证：okhttp3提供了更灵活的证书验证机制，支持自定义证书链验证。
4. 兼容性和稳定性
    + 更好的错误处理：okhttp3改进了错误处理机制，提供了更详细的错误信息和恢复策略。
    + 向后兼容：okhttp3在设计时考虑了向后兼容性，确保从okhttp迁移到okhttp3的平滑过渡。

#### QUIC、HTTP/1.1、HTTP/1.1+HTTP/2

+ QUIC是基于UDP的高性能协议，解决了TCP的队头阻塞问题，支持多路复用和快速连接建立，适合高延迟网络；
+ HTTP/1.1是传统的文本协议，不支持多路复用，性能较低但兼容性高；
+ HTTP/2在HTTP/1.1基础上引入二进制传输、多路复用和头部压缩，显著提升性能但仍依赖TCP
