
## 背景

引导业务是一个协议驱动、后台可配置的通用弹窗/浮层引导框架。业务方只需要给一个 placeId + triggerAction（触发动作），就能拉起一个带步骤（steps）、带按钮、带频控、带上报的引导层，UI 与内容完全由后台协议下发。核心入口是 GuideManager.java。

## 核心代码链路

+ 链路 1：主动触发（业务方直接调用 showGuideNow）
	+ 业务方调用 GuideManager.showGuideNow(activity, placeId, triggerAction, anchorPos)（见 CommFrameFactory.jumpToCommonGuide），进入两条分支：
		+ 数据未拉取：mGuideData 中没有该 placeId → 直接 startActivity(TvCommonGuideActivity)，Activity 内 GuideViewImpl.loadData() 同步拉取 loadGuide → GuideRequest → 网络请求 /pages/guide → parseJce 合并 dtReport → regroupGuides，再走 findGuide + loadGuideResources 预加载图片后上屏。
		+ 数据已拉取：tryShowGuide → findAndLoadGuide 命中已 ready 的 guide → 直接 startActivity 带上 guide_id，GuideViewImpl.loadResource() 秒开。
	+ 上屏后统一走 GuideContentViewModel.setGuideInfo 驱动 currentStepInfoData → handleStepChange 渲染 Poster/Button/倒计时；同时 FrequencyController.mark 打点频控。用户按键时 handleKeyEvent 调用 next() 步进，最后一步触发 dismissEvent → container.dismiss()，Activity onDestroy 时通过 EventBus.post(GuideCloseEvent) 通知业务方。

+ 链路 2：预加载（后台/静默）
	+ hasGuideForPlaceId(placeId) → false 时业务方主动 prepareGuide(placeId, TriggerAction.\_FOCUS_DOWN)
	+ 内部走 loadGuide → GuideRequest → regroupGuides → findAndLoadGuide → loadGuideResources（预下载图片到 Glide 内存）
	+ 用户真正触发时直接命中已 ready 的引导，tryShowGuide 秒开

+ 链路 3：Action 路由（服务端下发的 Action 触发）
	+ 后台通过 Action 下发 ACTION_ATTR_GUIDE_PLACE_ID / ACTION_ATTR_GUIDE_TRIGGER_ACTION
	+ CommFrameFactory.jumpToCommonGuide 方法解析后转调 GuideManager.tryShowGuide / showGuideNow
	+ 场景：ChildPurePlaylistUtils.java、ParentSettingContentDataUtils.java
