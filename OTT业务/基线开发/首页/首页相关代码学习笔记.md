
## 相关关键文件
### 基本文件介绍

+ HomeDataAdapter：首页数据适配器
+ HomeDataCenterServer：为adapter提供数据，处理网络请求、读取本地缓存等逻辑
+ NormalFragment：承载HomeDataAdapter
+ HomePageInfo：首页数据顶层的对象类型
+ JceStruct：首页的数据结构都需要继承这个类，他是网络传输的格式，到java中需要反序列化出来对象

### 一些工具类

+ MultiModeManager：多模式，常规、少儿、长辈
+ HomeLiteDataUtils：首页启动优化lite数据管理器
+ HomeDataUtils：首页相关数据处理工具类
+ HomeUrlUtils：首页请求URL拼接工具类
+ DrawableGetter：一些通用Drawable的getter和缓存（颜色、图标等）
+ UiTypeCompat：收拢UI图标调用
+ DesignUIUtils：定义了一些设计中公用的属性获取方法，减少UI更改的复杂度
+ FontSize、Common、Alpha、Padding等等
+ ViewDataParseUtils：根据ViewType解析ViewData并把解析结果赋值给原有的view结构


## 数据流
### HomeDataAdapter

关键类成员：
```java
private HomePageInfo mHomePageInfo // 存储首页所有数据
private ConcurrentHashMap<String, IHomePageDataListener> mNormalHomeDataListeners // 数据监听器
private HomeDataCenterServer mDataCenterServer // 数据服务中心
private Handler mWorkHandler // 工作线程Handler
private String mCurrentChannelId // 当前频道ID
```

核心方法：
1. 数据初始化
```java
public void init(int mode, int controlLevel, boolean preload, boolean sync)
private void doInit(int mode, int controlLevel, boolean preload)
```
2.  数据请求
```java
public void requestChannelFirstPageInfo(final String channelId)
private void requestChannelFirstPageInfoImpl(final String channelId, boolean needNetRequest)
public boolean requestChannelNextPage(final String channelId, String pageContext, boolean isEnd)
```
3.  数据通知
```java
private void notifyChannelPageUpdate(final ChannelStruct channelStruct, boolean isCache)
private void notifyChannelFirstPageError(...)
private void notifyChannelNextPageResult(...)
```
4.  数据合并处理
```java
private void mergeHomeFeedsData(ChannelPageInfo channelPageInfo, boolean isNeedNotifyEnd)
private ArrayList<SectionInfo> mergeChannelSectionsFirstPage(...)
```

### 数据获取

+ 从HomeDataAdapter开始：
	+ 数据加载流程：init() -> doInit() -> createDataCenter()
	+ 后续数据加载逻辑进入到 HomeDataCenterServer 类
	+ IHomeServerAdapter 首页Server回调接口（onInitFinish、onResponse、onError）
	+ HomeDataCenterServer 类中初始化的逻辑 initServer()从默认数据、缓存、网络请求中读取，但只第一次安装请求，其他情况不可用，是同步的
	+ requestServerDataAsync方法常用

### 数据刷新

HomeDataAdapter中的刷新部分：

+ notifyHomePageInfoGet
+ notifyChannelPageUpdate
+ notifyChannelFeedsChanged

### JceStruct数据结构

按照结构往下有一下这些类继承JceStruct：
+ HomePageInfo（ChannelInfo、BasicChannelInfo、ChannelInfoGroup）->
+ ChannelPageInfo（BackGroundPic）->
+ SectionInfo（VirtualControlInfo、DTReportInfo、ItemInfo、FreqLimitInfo等）->
+ GroupInfo（ItemInfo、DTReportInfo等）->
+ LineInfo（LineFillInfo、LineAsyncControlInfo等）->
+ ComponentInfo（DTReportInfo、FreqLimitInfo等）->
+ GridInfo（Value、DTReportInfo、GridLayout等）->
+ ItemInfo（Value、ReportInfo、Action等）->
+ View、Value

> View中mData属性类型为JceStruct，其需要依靠ViewDataParseUtils解析viewData，他会被解析成其他类型，比如HeadPosterPlayerViewInfo等
> 
> Value则涵盖了六种类型，往下再无其他结构
