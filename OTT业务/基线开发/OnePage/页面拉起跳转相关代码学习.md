

## 概述

整体链路：
1. Action / actionId / 参数
2. FrameManager 统一跳转入口
3. CommFrameFactory 按 actionId 分发
4. 拉起 Activity 或复用已有 OnePageActivity
5. OnePageActivity 根据 actionId 创建 Fragment
6. OnePageFragmentFactory 反射调用 Fragment.newInstance(Bundle)


几个核心类/概念：

| 概念                     | 理解                                  | 典型代码位置                             |
| ---------------------- | ----------------------------------- | ---------------------------------- |
| Action                 | 数据协议里的跳转动作，通常来自服务端或业务 ViewModel     | 各种 ItemInfo.action、Action.actionId |
| actionId               | 页面路由 ID，比如详情页、首页、半屏页                | ActionId.java                      |
| ActionValueMap         | 跳转参数 Map，里面放 cid、vid、tabId、来源参数等    | ActionValueMap.java                |
| FrameManager           | 全局跳转门面，所有跳转尽量汇聚到这里                  | FrameManager.java                  |
| CommFrameFactory       | 具体跳转实现，按 actionId 分发到不同页面           | CommFrameFactory.java              |
| OnePageActivity        | 一页式页面容器，里面承载各种 BaseOnePageFragment  | OnePageActivity.java               |
| OnePageFragmentFactory | 根据 actionId 创建具体 Fragment           | OnePageFragmentFactory.java        |
| OpenJumpAction         | 外部拉起、Scheme 跳转的核心处理类                | OpenJumpAction.java                |
| OpenJumpParser         | 把外部 URI / Intent 解析成 OpenJumpAction | OpenJumpParser.java                |
