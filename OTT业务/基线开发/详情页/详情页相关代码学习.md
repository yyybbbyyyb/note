
## OneDetailCoverPageFragment

#### 概述

OnePage框架-专辑详情页容器
+ 详情页外壳
+ 负责接收参数、初始化播放模型、加载真正的详情页内容 Fragment
+ 负责生命周期、上报、EventBus、半屏返回、语音、播放器窗口状态等外围逻辑

#### 继承
+ `extends BaseOneDetailPageFragment`
	1. 详情页根布局创建: `NewArchDetailViewFactory.getOneDetailPageContentViewBinding();`
	2. 播放器窗口状态监听
	3. 大同 pageId/pageParams 初始化
	4. 半屏容器层级处理
	5. 返回、语音、播放器全屏/小窗状态处理
	6. `hasPageFragment()` 判断子详情页是否已加载

+ implements ICoverDetailContentIdProvider
	1. 提供详情页内容 ID：`getContentId`

+ implements IOnePageLifecycleOwner
	1. OnePage页面生命周期的持有者接口

#### 关键字段
+ `mControlInfo`：详情页接口下发的专辑信息
+ `mPageFragmentLoad`：页面内容是否已经加载
+ `mNewArchDetailCoverViewModel`：存储以下核心数据：
	+ `mCoverControlInfo`：专辑控制信息
	+ `mListData`：详情页列表内容数据，右侧工具栏/选集/推荐等都在这里
	+ `DetailHeaderInfo`：头部/左侧信息区相关数据

#### 关键方法
**onCreate**
+ DetailCoverPlayHelper.install(this)：数据拉取主链路的关键入口之一
	+ OneDetailCoverPageFragment.onCreate
	+ DetailCoverPlayHelper.install(this)
	+ 创建/获取 DetailCoverPlayModel
	+ playModel.setArguments(...)
	+ 创建详情页数据模型并开始数据链路
+ OnePage 生命周期

**onViewCreated**
真正加载内容页的位置
```java
if (!hasPageFragment()) {
    loadArguments();
    if (getNewArchDetailCoverViewModel().isInChildModeAutoFullScreen()) {
        TVCommonLog.w(mTag, "onViewCreated: auto full screen! load page later");
    } else {
        loadPageFragmentIfNeed();
    }
}

getNewArchDetailCoverViewModel()
        .getControlInfoLive()
        .observe(getDataLifecycleOwner(), coverControlInfo -> mControlInfo = coverControlInfo);
```
+ loadArguments()：这个方法不是发网络请求，更多是处理：
	+ 获取 cid
	+ 设置上报参数
	+ 设置来源参数
	+ 设置广告归因
	+ 设置大同 page params
+ loadPageFragmentIfNeed()：加载真正 UI 的地方
	+ `fm.beginTransaction().add(FRAGMENT_CONTAINER_VIEW_ID, new DoubleRowDetailCoverPageFragment(), FT_PAGE);`
+ 缓存ControlInfo到OneDetailCoverPageFragment中

#### 数据监听
```java
// DetailCoverPlayHelper.java
final NewArchDetailCoverViewModel model = getNewArchDetailCoverViewModel();

mPlayHelper.observe(IDetailListData.class, listData -> {
    model.setListData(listData);
});

// DetailInfoModel.java
@CallSuper
public void registerModelInput(@NonNull final ModelRegistry registry) {
	registry.addModelSource(IDetailListData.class, getDetailListData());
}
```

数据从CoverDetailRootModel中请求，回包后经过多重监听，数据经过DetailCoverInfoModel、NewArchDetailCoverViewModel最终到DoubleRowDetailCoverPageFragment

#### 关键类

**NewArchDetailCoverViewModel**作为数据上屏前的中转站
```java
// NewArchDetailCoverViewModel.java
public LiveData<ListData> getListDataLive() {
    return mSmartListDataLive;
}

combine(mListDataLive, mPlayerVisible, mListDataInUseLive, mUIBusyLive, ...)
```
在NewArchDetailCoverViewModel中ListData经过过滤/节流，返回的是SmartListData



**DoubleRowDetailCoverPageFragment**是真正的上屏入口
```java
// DoubleRowDetailCoverPageFragment.onCreateView()
final NewArchDetailCoverViewModel model = getViewModel();
filterEqual(model.getListDataLive()).observe(getDataLifecycleOwner(), this::setListDataToAdapter);
model.getControlInfoLive().observe(getDataLifecycleOwner(), this::setControlInfo);
model.getDetailHeaderInfoLive().observe(getDataLifecycleOwner(), this::handleHeaderInfo);
delay(model.getPlayerVisibleLive()).observe(getDataLifecycleOwner(), this::onPlayerVisibleChanged);
```
+ ListData：setListDataToAdapter()（右侧列表/全屏提示列表上屏）
	+ setListDataToAdapter分了一下三种情况：
		+ 空数据
		+ Loading/Error/无版权等非内容提示
		+ 正常内容数据
	+ setListDataToSideAdapter：右侧列表拆成上半区和下半区
+ CoverControlInfo：setControlInfo()（付费状态、上报参数等）
+ DetailHeaderInfo：handleHeaderInfo()（左侧详情信息/头部信息）
+ PlayerVisible：onPlayerVisibleChanged()（影响播放器和内容布局）




## 详情页相关重要类

| 类名                                                                                              | 关键作用                                                                                                                                                                                        |
| ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| OneDetailCoverPageFragment                                                                      | 详情页外层壳/调度器。负责接收 arguments、解析 cid/vid/actionId 等页面参数，安装 `DetailCoverPlayHelper`，创建/管理真正的详情页内容Fragment（如`DoubleRowDetailCoverPageFragment`）。它不是主要画UI 的类，但它决定详情页怎么启动、数据模型怎么接入。                 |
| DoubleRowDetailCoverPageFragment                                                                | 详情页真正的双列 UI 主体。负责左侧播放器/信息区、右侧列表、工具栏、半屏浮窗入口等 UI 上屏逻辑。setListDataToAdapter()、setListDataToSideAdapter() 处理数据显示到页面上。                                                                           |
| NewArchDetailCoverViewModel                                                                     | 详情页 UI 层 ViewModel。承接 DetailCoverPlayHelper 推过来的 ListData、CoverControlInfo、DetailHeaderInfo 等数据，再通过 LiveData 暴露给 DoubleRowDetailCoverPageFragment。它是数据从播放/模型层进入 UI 层的核心中转站。                 |
| DetailCoverPlayHelper                                                                           | 详情页播放/数据桥接 Helper。内部通过 mPlayHelper.observe(...) 监听 IDetailListData、ICoverControlInfo、IDetailHeaderInfo 等通道，然后调用 NewArchDetailCoverViewModel.setListData()、setControlInfo()、setHeaderInfo()。 |
| DetailCoverPlayModel                                                                            | 详情页播放模型/页面模型容器。负责承载详情页播放相关状态、参数、播放控制、数据模型绑定。它本身不是最终请求类，但它持有 ModelRegistry，是数据模型和 UI Helper 之间的重要容器。                                                                                         |
| DetailCoverInfoModel                                                                            | 详情页专辑数据模型聚合层。内部持有真正的详情数据模型，例如 mDetailModel，并在 registerModelInput() 中把 CoverControlInfo、DetailHeaderInfo、StatusBarAdKey、PrePlaying 等 LiveData 注册到 ModelRegistry。                             |
| DetailInfoModel                                                                                 | 详情页基础数据模型父类。负责注册通用详情数据，例如 IDetailListData、IPlaylistCollection、IPlayerCardButton。其中 getDetailListData() 是右侧列表数据的关键来源。                                                                        |
| CoverDetailRootModel                                                                            | 真正承接详情接口数据的 Root Model。负责详情页接口请求、解析、转换，并产出 LiveData\<ListData\>、LiveData\<DetailHeaderInfo\>、LiveData\<CoverControlInfo\> 等数据。                                                              |
| ModelRegistry                                                                                   | 播放模型的数据通道注册表。用 Class 作为 key，把类似 IDetailListData.class、ICoverControlInfo.class 映射到具体 LiveData。数据模型通过 addModelSource() 注册数据源，UI 侧通过 observe() 监听。                                             |
| ModelObserverManager                                                                            | ModelRegistry 的观察者管理器。OnePlayHelper.observe(...) 最终会走到它的 getModelOutput(clazz).observe(observer)，负责把某个接口类型对应的 LiveData 插到观察者上。                                                              |
| OnePlayHelper                                                                                   | 通用播放 Helper 封装。DetailCoverPlayHelper 内部使用它来连接 DetailCoverPlayModel 和生命周期，并暴露 observe(Class\<M\>, Observer\<T\>) 这种按接口类型监听数据的能力。                                                             |
| CommonModel\<T\>                                                                                | 模型数据通道接口。例如 IDetailListData extends CommonModel\<ListData\>，代表这个通道输出的是 LiveData\<ListData\>。                                                                                                |
| IDetailListData/<br>ICoverControlInfo/<br>IDetailHeaderInfo/<br>IStatusBarAdKey/<br>IPrePlaying | 详情页列表数据通道标识。<br>详情页控制信息通道标识。<br>详情页头部/基础信息通道标识。<br>状态栏广告 key 通道标识。<br>预播放状态通道标识。                                                                                                            |
| ListData                                                                                        | 详情页右侧列表上屏的核心数据结构。DoubleRowDetailCoverPageFragment 拿到它后，会通过 ListData.splitListData(...) 拆成右侧上半区和下半区。                                                                                         |
| DetailHeaderInfo                                                                                | 详情页头部信息结构体。承载标题、简介、封面、标签等信息，主要对应左侧详情信息区。                                                                                                                                                    |
| CoverControlInfo                                                                                | 详情页控制信息结构体。承载控制态数据，例如播放权限、按钮状态、会员/试看/追剧等相关状态。                                                                                                                                               |
| OnePageActivity                                                                                 | 承载 OnePage Fragment 的 Activity。从 Intent 里取 actionId 和 extras，然后交给 OnePageFragmentFactory 创建具体 Fragment。                                                                                     |
| OnePageFragmentFactory                                                                          | 页面 Fragment 工厂。根据 actionId 判断当前要创建详情页、首页、列表页还是其他页面。                                                                                                                                         |
| BaseOneDetailPageFragment                                                                       | 详情页 Fragment 基类。提供详情页通用能力，例如 getContentId()、页面生命周期、数据引擎释放、路径名等。                                                                                                                             |
| ConstantsAndUtils                                                                               | 详情页工具方法集合。用于从 arguments / OpenJumpAction.EXTRA_DATA 里取 coverId，也包含一些半屏拉起、Action 解析等辅助能力。                                                                                                    |
| OpenJumpAction / <br>ActionValueMap                                                             | 跳转参数承载结构。详情页的 cid、vid、来源、指定播放参数等通常藏在 OpenJumpAction.EXTRA_DATA 对应的 ActionValueMap 里。                                                                                                        |
