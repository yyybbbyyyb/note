
依赖相关：
+ build-ext.gradle
	+ 这个和插件没啥关系，单纯就是依赖可以规范的写在这里：buildDeps.stetho

注册插件：

+ Plugin.java
	+ 获得插件实例的代码
	+ 实现IRdefensePerformer rdefensePerformer()和void rdefensePerformer(@NonNull LoadCallback callback)方法（我理解是一个是同步的，一个是异步支持回调的）
+ PluginLauncher.java
	+ SDK插件启动器
+ AveLoader.java
	+ 更底层的插件加载框架，调用vPlugin
+ PluginUtils.java
	+ 插件化的工具类，里面定义了一些常量，也有一些工具方法，定义一个MODULE_\*常量，并且在最下面isAvePlugin中添加判断逻辑
+ plugin_list.json
	+ 注册插件信息，这里也可以配置远程下载apk

插件实现：

+ Plugin_Base/src/com/tencent/qqlivetv/plugincenter/perform
	+ 定义插件的I\*Performer接口
+ proj.android/src/com/tencent/qqlivetv/plugin/perform
	+ 插件的no-op实现（Default实现）
+ 根目录下具体的Plugin_*
+ 实现一个application，即插件的具体逻辑。至少需要有一个Performer来实现上面定义的I\*Performer接口

插件编译：

+ app/build.gradle
	+ 该文件下添加`mergetTask.dependsOn "buildStethoPlugin${buildType}"`
	+ 并且`isDebugPluginOpen()`开成true
+ plugin_debug.gradle
	+ 在这里注册相应的task
+ proj.android/patch/
	+ 这里实现一个*.xml提供版本号等信息
+ gradle.properties
	+ 这里可以定义版本号，插件版本等信息

> 更加详细的怎么添加一个新的插件可以参考：[[stetho插件化设计文档]]
