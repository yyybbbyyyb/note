### 频控代码

| 类                          | 说明                                              |
| -------------------------- | ----------------------------------------------- |
| FrequencyController        | 通用频控总入口（isLimited / mark / clearAll）            |
| ClientFrequencyController  | 面向"全局配置(ConfigManager)"的频控封装，业务侧一般直接用           |
| FrequencySessionHelper     | 会话级频控辅助                                         |
| FrequencyDimenKeyGenerator | 维度Key生成器（例如按用户、按设备、按VIP等生成不同key）                |
| FreqLimitReporter          | 频控上报器（配合后台的 uniqueKey 去重上报）                     |
| ILimiter                   | 限频器接口：isLimited(key) / mark(key)                |
| Limiters                   | 工厂：根据后台策略选择具体限频器（含 NoLimiter / LimitEverything） |
| TimeLimiter                | 分/时/天时间窗口                                       |
| FrequencyRecorder          | MMKV 存储封装                                       |
+ 一次限频判断的典型链路
	+ 业务侧拿到后台下发的 FrequencyLimiters，调用 FrequencyController.isLimited(limiters) 或 mark(limiters)。
	+ FrequencyController 遍历 limiters → 每个 FrequencyLimiter → 每个 FrequencyLimitStrategy，用 limiter_key + strategy 做判断/标记。
	+ Limiters.getLimiter(strategy) 根据 strategy.intervalType（分/时/天/周/月/会话/版本/APP 生命周期）选一个具体的 ILimiter 实现。
	+ 具体限频器通过 FrequencyRecorder（MMKV）或 FrequencySessionHelper（内存）持久化/读取展示记录，判断是否超限。
+ 特殊入口 ClientFrequencyController
	+ 它是给"客户端本地基于 ConfigManager 配置做频控"的场景准备的：读取一个 JSON 配置（ClientFrequencyConfig），支持维度化 Key（用户/设备/VIP 等），最终仍然把请求转发给FrequencyController.mark/isLimited，只是入参形态不同。

### 全局配置

+ ConfigManager
	+ 基线中配置管理器
	+ optGetConfig读取配置的核心方法，依次从remoteMap、lastMap、MMKV中读取，最后兜底找localManager
	+ getRemoteConfig方法利用ConfigRequest、ConfigResponse请求远程配置信息，并且存储进remoteMap、MMKV

+ 刷新机型能力配置函数调用链：
	+ DeviceFunctionManager.refreshDevFuncConfig：从DeviceFunctionSettingIndex中获得所有的配置的Key
	+ ConfigRepo.loadRemoteConfigs：提供异步线程池以及回调函数
	+ ConfigFetcher.loadRemoteConfigs：key按照最大长度分段，利用线程执行请求发起
	+ DevConfigRequestProxy.sendConfigRequest：经过漫长的调用TVDevTypeService、TVDevType、DevTypeV1、DevCapRequestV1...，请求成功后执行回调
	+ ConfigRepo.updateConfigToRepo：更新配置，分别更新到流水线与具体字段里
	+ ConfigPipeline.updateConfig：更新到流水线里
	+ ConfigRepo.updateConfigToValueField：更新到具体字段里
