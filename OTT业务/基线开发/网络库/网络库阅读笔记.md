
## 概述

本人深入阅读TvNetworkLib网络库代码，侧重于跟踪网路请求如何被传递处理返回，关注架构设计，将网络库架构分为一下几部分：
![[网络库阅读笔记-1.png]]

+ 基线初始化部分为基线内代码，基线引入ottnetworkservice依赖，在InterfaceTools中初始化ITVNetworkService，并且利用其提供的get、getOnSubThread等方法传递ITVRequest；同时TvNetworkConfigManager配置也通过接口传递到网络库中
+ 全局服务部分代码位于ottnetworkservice库中，主要负责参数加载、服务启动等功能，实际功能是为基线提供调用接口，并且封装一层GlobalManager的服务
+ 请求管理部分代码位于library中，是volley库范畴，主要功能是初始化服务引擎（AppEngine），处理超时和缓存等逻辑
+ 队列调度部分代码也位于library中，隶属volley库，主要负责异步/同步处理请求，Executor线程池架构，高效动态调度处理网络请求
+ 网路协议部分代码同上，主要负责协议选择，相应相应的处理，同时将HttpResponse包装为 NetworkResponse返回
+ 协议栈工厂部分代码同上，主要负责协议管理、错误处理、URL适配，此时已经使用具体的网络库来处理请求，利用client创建call，同时包装Response为HttpResponse返回

## 具体类简要解析

### 请求管理

+ GlobalManager.java
	+ 工具类去初始化RequestQueue、AppEngine等
+ AppEngine.java
	+ 用于处理网络请求的辅助类，主要用于从远程URL加载数据。
	+ TimeOutRunnable，处理请求超时逻辑，优先从缓存中返回数据。
	+ getResponseSync同步处理请求，get和getResponseImpl处理异步请求，异步请求中有超时缓存相关逻辑，用到的结构是Handler
	+ setCallback服务于异步请求，设置成功和失败的回调，失败有读取缓存的操作
	+ sendCacheRequest发送缓存中的请求

### 队列调度

+ Cache.java（volley）
	+ Entry条目，toolbox中有具体的实现的cache
+ ThreadPoolRequestQueue.java
	+ RequestQueue的具体实现，包括：executeSync、add等抽象方法的具体实现
	+ executeSync的实现主要是预处理请求后，通过createRequestJob创建networkJob
	+ add的实现是利用不同的模式选择不同的线程池来处理RateLimitNetworkJob
	+ CombinedJob：CacheAndNetworkJob、RateLimitNetworkJob，rate有限流逻辑，涉及到RateLimitManager；cache从缓存中读取，并判断是否需要刷新
	+ 分功能设立不同的线程池：mExecutor、mHighPriorityExecutor、mLowPriorityExecutor等等；还分线程池模式
	+ promotePendingRequests动态调度等待队列中的请求
+ RequestQueue.java
	+ executeSync处理同步请求，返回Response
	+ add处理异步请求
	+ prepareRequest预处理，将请求添加到mCurrentRequests中，配置请求的时间戳等设置，进入网络库准备排队
+ NetworkJob.java
	+ 根据请求类型（网络请求或缓存写入）调用对应方法
	+ requestOperation处理网路请求，通过Network接口执行请求：mNetwork.performRequest(request);

### 网络协议

+ Volley.java
	+ newRequestQueue这个方法初始化requestQueue，这个地方会新建network，并且传递HttpStack
	+ 主要的职责就是新建RequestQueue，并且选择配置，配置还会同步到OkHttpClientBuilderSetting中
+ BasicNetwork.java
	+ 继承自Network，实现performRequest方法
	+ performPequest中有选择http栈的逻辑，用http栈的performRequest获得HttpResponse，然后包装返回NetworkResponse，注意还调用HttpStackManager.getInstance().onSuccess(request, request.getHttpStackType());
	+ getHeaders、 addCacheHeaders、parseProtocol、entityToBytes等众多工具方法
+ MultiStackNetwork.java
	+ 重写了selectHttpStack方法，可以根据request中的已有字段选择协议以及http栈（也用到了一些判断NewOkHttpStack是否可用的方法）

### 协议栈工厂

+ HttpStackManager.java
	+ 单例初始化时注册 StackHolder，mDefaultStackHolder 为 legacy_okhttp
	+ defaultStack、get方法获得HttpStack
	+ initByType(HttpStackType)：根据类型初始化HTTP栈，并检查其可用性（兜底能力）
	+ 错误处理和成功回调方法（onRetry、onVolleyError、onSuccess）。
	+ 整体上类似于一个HttpStack的工厂模式
+ HttpStack.java
	+ 接口：HttpResponse performRequest(Request\<?\> request, Map<String, String> additionalHeaders)
	+ TQuicStack、OkHttpStack、OkHttpStack2实现
+ TypedHttpStack.java
	+ operateRequestUrl方法处理请求的URL，同时区分类型
	+ 实现HttpStack接口，主要用于封装类型
+ OkHttpStack.java
	+ 继承自TypedHttpStack，实现performRequest方法
	+ 利用com.ktcp.tencent.okhttp3.OkHttpClient，创造okHttpRequestBuilder等操作，利用client实现调用
	+ TQuicStack、OkHttpStack2类似，只有细微区别

### 全局服务

+ TvGlobalManager.java
	+ 配置host、port等配置后实例化一个GlobalManager（相当于对他的封装），也提供相同的配置方法
	+ 也提供了get方法，入参ITVRequest、ITVResponse适配成AppResponseHandler、BaseRequestHandler，也有getOnMainThread方法
+ ITVNetworkService.java
	+ 网络库服务接口，TVNetworkService具体实现，提供发起网路请求（绑定activity、fragment）等方法
	+ 具体实现调用的是TVNetwork中的get方法
+ TVNetwork.java
	+ 利用TvGlobalManager中的方法去提供get等方法让Service透传
	+ 在这里会初始化TvGlobalManager（工厂模式GlobalManagerFactory），这里倒是提供了很多set方法来配置TV GlobalManager
+ TvNetConfigDefault.java
	+ 实现接口ITvNetConfig，可以设置，靠基线传参过来

### 基线初始化

+ TvNetworkConfigManager.java
	+ 利用TvNetworkConfig，该config继承TvNetConfigDefault
+ InterfaceTools.java
	+ 实例ITVNetworkService
+ TaskInitNetWork
	+ InterfaceTools初始化利用到这个task，并且传递了配置等信息

### 数据结构

+ Response.java
+ HttpResponse.java
+ NetworkResponse.java
+ Request.java
	+ LoadMode：SERVER_AND_CACHE、SERVER
	+ LogicTimeOutMode：LOGIC_TOUT_MIDDLE
	+ requestHandler.addMarker("cache-hit"); requestHandler.markDelivered();
	+ requestHandler.getRealProtocol(); requestHandler.getAlterDataSource()
	+ request.initTimestamp(); request.setRequestQueue(this);
	+ request.setAddRecord(new Exception());（用这个Exception记录Request进入网络库的堆栈，出现异常无法根据url等信息时用于跟踪来源）
	+ request.setSequence(getSequenceNumber());、request.addMarker("add-to-queue");
	+ initSpan()
	+ request.isSingleThreadMode()，request.shouldCache()，request.checkFromCache()
+ ITVRequest.java
	+ TVJceRequest、TVCommRequest、TVSimpleRequest三个继承者
+ BaseRequestHandler.java
+ AppResponseHandler.java
	+ onFailure、onSuccess

### 工具

+ RateLimitManager.java
	+ 限流器
+ RespProperty.java
	+ 打包Response的响应
+ SyncRequestDelivery.java
	+ 同步请求使用的Delivery，功能为保存Response，执行完成后可以取出来
+ ExecutorDelivery.java
	+ 异步请求使用的Delivery
+ CgiAccessQualityData.java
	+ 记录和上报网络请求的质量数据
+ HttpStackType.java
	+ UNKNOWN、LEGACY_OKHTTP、OKHTTP、TQUIC，四种类型
+ HttpDNSInterceptor.java
	+ TvGlobalManager.createHttpDnsInterceptor中创建，实际上是适配器模式，真实传入类型为ITVHttpDNSInterceptor，最终到具体的HttpStack中还要适配成dns
	+ 用于在 DNS 解析阶段拦截并替换默认的域名解析逻辑
+ RequestProxyFactory.java
	+ 提供ITVRequest做转换的方法
